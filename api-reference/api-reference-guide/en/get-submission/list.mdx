<Warning>
주의사항
1. 이 API 호출은 모든 제출 건을 반환하므로 대량의 데이터가 포함될 수 있습니다.
2. 성능 최적화를 위해 페이지네이션 파라미터를 사용하는 것이 좋습니다.
</Warning>

## 1. 엔드포인트

```text GET/Submission/submissionId
GET https://rest-api.argosidentity.com/v3/submission
```
## 2. 인증

x-api-key 헤더에 API 키를 포함시켜야 합니다:

```text x-api-key
x-api-key: {yourAPIKey}
```


## 3. 요청 파라미터
요청 본문은 JSON 형식이어야 합니다. 다음은 필드 설명입니다:

<ResponseField name="admin" type="string" required="true">
  조회할 제출 건의 고유 ID
</ResponseField>


## 4. 요청 예시

```curl POST/Submission
curl --location --request GET 'https://rest-api.argosidentity.com/v3/submission?submission_id={submission_id}' \
--header 'x-api-key: {yourAPIKey}'
```
