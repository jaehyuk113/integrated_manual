<Update label="2025-05-08">
  # Custom DI 제공

  관리자는 Custom DI 를 Submission 에서 제공되는 정보를 기반으로 설정할 수 있습니다. 설정된 Custom DI 로 관리자는 중복 관리를 할 수 있습니다. API 으로는 GET/SUBMISSION 을 통해 custom DI 를 조회할 수 있습니다.

  # URL Query 암호화 옵션 allowedIdTypes, allowedDocumentFields 추가
    신분증 기반의 KYC 인 ID document 프로세스가 진행되는 경우, 허용할 신분증 타입과, 허용할 필드를 설정할 수 있습니다. URL Query 에 encrypted 파라미터로 암호화하여야 설정이 가능합니다.

  # Projection

  GET/SUBMISSION 시 노출되는 개인의 민감한 정보들은 Projection 을 설정하여 노출되지 않게 설정하실 수 있습니다. Projection 목록은 GET/Projection, POST/Projection, and DELETE/Projection로 관리하실 수 있으며, Projection 적용은 사용자가 KYC 진행시 입력하는 URL 의 Query String 옵션을(Encrypted 필수) 사용하여 적용하실 수 있습니다.

</Update>

<Update label="2025-03-27">
  # Knowledge-based 추가

  기존의 LiveForm 은 ID Document 로 정의하고, 신분증 없는 신원확인을 위한 Knowledge-based 기능을 추가하였습니다. 관련해서 매뉴얼의 암호화, Query String 관련 업데이트를 하였습니다.
</Update>

<Update label="2024-01-23">
  # DI 필드 추가

  - `GET/Submission API`: 중복 데이터를 식별하고 관리할 수 있는 DI(duplicated information) 필드가 추가되었습니다. 이 필드는 중복된 정보의 해시값을 제공합니다.
  - `Webhook Events`: 모든 이벤트에 DI(duplicated information) 필드가 추가되어, 중복된 정보에 대한 해시값을 반환합니다.
</Update>

  # 개발 문서가 새로워졌습니다 🚀

  더 쉽고 직관적인 개발 경험을 위해 개발 문서를 전면 개편했습니다. 처음 사용하는 사용자부터 숙련된 개발자까지, 누구나 빠르게 이해하고 활용할 수 있도록 준비했어요.

  ## 이번 개편에서 달라진 점

  ### 1. 문서 구조를 새롭게 정비했습니다

  더욱 체계적이고 직관적인 구조로 문서를 재구성했습니다.
  필요한 정보를 빠르게 찾아볼 수 있도록 목차와 섹션을 깔끔하게 정리했어요.

  ### 2. API 레퍼런스를 강화했습니다

  이제 모든 엔드포인트에 상세한 설명과 함께 실제 예제 코드를 제공합니다.
  매개변수와 반환값도 한눈에 파악할 수 있어 개발 효율이 높아집니다.

  ### 3. 튜토리얼 섹션이 추가되었습니다

  처음 사용하는 사용자도 쉽게 따라할 수 있도록 단계별 가이드를 제공합니다.
  기초부터 시작해 문서를 활용하는 방법을 차근차근 배워보세요.

  ### 4. 검색 기능을 업그레이드했습니다

  원하는 정보를 더 빠르게 찾을 수 있도록 검색 알고리즘을 개선했습니다.
  키워드 자동 완성과 관련 문서 추천 기능도 추가되었어요.
