## 1. 엔드포인트

```text GET/Submission
GET https://rest-api.argosidentity.com/v3/submission
```
## 2. 인증

x-api-key 헤더에 API 키를 포함시켜야 합니다:

```text x-api-key
x-api-key: {yourAPIKey}
```


## 3. 요청 본문
요청 본문은 JSON 형식이어야 합니다. 다음은 필드 설명입니다:


<ResponseField name="submissionId" type="string">
 특정 제출 건의 고유 ID
</ResponseField>

<ResponseField name="userid" type="string">
 사용자 ID와 일치하는 목록 조회
</ResponseField>
<ResponseField name="email" type="string">
 이메일 주소와 일치하는 제출 목록 조회
</ResponseField>
<ResponseField name="count" type="string">
 반환할 결과의 수 (기본값: 50)
</ResponseField>

<ResponseField name="nextpage_id" type="string">
 다음 페이지 조회를 위한 ID
</ResponseField>
<ResponseField name="nextpage_date" type="string">
 다음 페이지 조회를 위한 날짜
</ResponseField>
<ResponseField name="start_date" type="string">
 조회 시작 날짜 (YYYY-MM-DD)
</ResponseField>
<ResponseField name="end_date" type="string">
 조회 종료 날짜 (YYYY-MM-DD) 
</ResponseField>
<ResponseField name="request_type" type="string">
 	request_type 파라미터를 사용하여 특정 데이터 유형만 선택적으로 조회할 수 있습니다. 다음 옵션 중 하나를 선택하여 입력할 수 있습니다:
    - `kyc`: KYC 관련 데이터와 submission_id를 반환합니다.
    - `aml`: AML 관련 데이터와 submission_id를 반환합니다.
    - `data`: 제출자의 개인 정보 데이터와 submission_id를 반환합니다.
    - `others`: KYC, AML, 개인 정보를 제외한 나머지 모든 데이터 항목을 반환합니다.
</ResponseField>

## 4. 요청 예시

```curl POST/Submission
curl --location 'https://rest-api.argosidentity.com/v3/submission?submission_id={submission_id}' \
--header 'x-api-key: {yourAPIKey}'

```

### 5-3. 오류 코드
| 에러 코드                | 메시지  | 설명 |
|-------------------------|------| ---- |
| `invalid_payload`      | Invalid payload | 요청 본문을 파싱할 수 없음 |
| `missing_data`         | Required input data is missing | 필수 필드가 누락됨 |
| `invalid_project`      | Invalid project | 프로젝트 ID가 유효하지 않음 |
| `invalid_admin`        | Invalid admin | 관리자 계정에 이 프로젝트에 대한 권한이 없음 |
| `invalid_parameter`     | invalid parameter: `{parameter}` | 예상치 못한 매개변수가 제출됨 |
| `invalid_format`       | invalid format: `${parameter}` | 매개변수가 예상 형식과 일치하지 않음 |
| `processing_error`      | Failed to complete migration | 처리 중 알 수 없는 오류 발생 |




## 6. 암호화 옵션
보안 강화를 위해 요청 본문을 암호화할 수 있습니다. 암호화를 사용할 때:
전체 요청 본문 객체를 암호화합니다.
암호화된 문자열을 `body` 매개변수로 전송합니다.

### 6-1. 암호화된 요청 예시

```json result.json
body : encrypt({
		email : 'string',
		fullName : 'string',
		issuingCountry : 'string',
		birthDate: 'string'
		...
	})

```
### 6-2. 암호화된 응답
응답에는 isEncrypted 플래그와 암호화된 데이터가 포함됩니다:

```json result.json
response : {
   body : {
    "data": "encrypted-string",
    "isEncrypted": true
   }
}

```
