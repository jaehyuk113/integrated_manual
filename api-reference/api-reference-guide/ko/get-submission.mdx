## 1. 엔드포인트

```text GET/Submission
GET https://rest-api.argosidentity.com/v3/submission
```

## 2. 인증

`x-api-key` 헤더에 API 키를 포함시켜야 합니다:

```text x-api-key
x-api-key: {yourAPIKey}
```

## 3. 요청 파라미터

요청 파라미터는 URL query 형식이어야 합니다. 각 파라미터에 대한 설명은 다음과 같습니다.

<ResponseField name="submission_id" type="string">
  특정 제출 건의 고유 ID
</ResponseField>

<ResponseField name="userid" type="string">
  사용자 ID와 일치하는 목록 조회
</ResponseField>

<ResponseField name="email" type="string">
  이메일 주소와 일치하는 제출 목록 조회
</ResponseField>

<ResponseField name="count" type="string">
  반환할 결과의 수 (기본값: 50)
</ResponseField>

<ResponseField name="nextpage_id" type="string">
  다음 페이지 조회를 위한 ID
</ResponseField>

<ResponseField name="nextpage_date" type="string">
  다음 페이지 조회를 위한 날짜
</ResponseField>

<ResponseField name="start_date" type="string">
  조회 시작 날짜 (YYYY-MM-DD)
</ResponseField>

<ResponseField name="end_date" type="string">
  조회 종료 날짜 (YYYY-MM-DD)
</ResponseField>

<ResponseField name="request_type" type="string">
  request_type 파라미터를 사용하여 특정 데이터 유형만 선택적으로 조회할 수 있습니다. 다음 옵션 중 하나를 선택하여 입력할 수 있습니다:

  - `kyc`: KYC 관련 데이터와 submission_id를 반환합니다.
  - `aml`: AML 관련 데이터와 submission_id를 반환합니다.
  - `data`: 제출자의 개인 정보 데이터와 submission_id를 반환합니다.
  - `others`: KYC, AML, 개인 정보를 제외한 나머지 모든 데이터 항목을 반환합니다.
</ResponseField>

## 4. 요청 예시

```curl POST/Submission
curl --location 'https://rest-api.argosidentity.com/v3/submission?submission_id={submission_id}' \
--header 'x-api-key: {yourAPIKey}'
```

## 6. 암호화 옵션

<Note>
  사전 요구 사항 : 안전한 암호화 옵션을 활성화해야 합니다. 암호화 적용 전에 [가이드](/getting-started/ko/encrypt-and-decrpyt-data/overview#3)를 먼저 확인해주세요.
</Note>

보안 강화를 위해 요청 본문을 암호화할 수 있습니다. 암호화를 사용할 때:
전체 요청 본문 객체를 암호화합니다.
암호화된 문자열을 `body` 매개변수로 전송합니다.

### 6-1. 암호화된 요청 예시

```json
{
  "start_date": "2025-01-01",
  "end_date": "2025-03-01"
}
```

안전한 데이터 전송 옵션이 활성화 된 프로젝트에서 줄 수 있는 옵션은 다음과 같습니다.

```json encrypt-available-params.json
body : encrypt({
		email : 'string',
		start_date : 'string',
		end_date : 'string',
		submission_id: 'string'
	})
```

위 값을 [AES-256-ECB](https://developers.argosidentity.com/getting-started/en/encrypt-and-decrypt-data/overview#3-2-api-request-data-encryption-aes-256-ecb) 로 암호화 한뒤 URI encode 를 합니다.

```
N34SNtWaavEfgtg1g%2Bo%2B9JhQ9rp9dGUbyFNxAsHKKGH24aVQTRXYfNpFDHIGJU6Wo0RVpOupAubiDvFDuFyTkw%3D%3D
```

그리고 Query String 에서`data` 파라메터로 위 암호화된 값을 입력 후 호출합니다.

```bash
curl -X GET "https://rest-api.argosidentity.com/v3/submission?data=N34SNtWaavEfgtg1g%2Bo%2B9JhQ9rp9dGUbyFNxAsHKKGH24aVQTRXYfNpFDHIGJU6Wo0RVpOupAubiDvFDuFyTkw%3D%3D" \
--header 'Content-Type: application/json' \
--header 'x-api-key: change_your_api_key'
```

### 6-2. 암호화된 응답

응답에는 isEncrypted 플래그와 암호화된 데이터가 포함됩니다:

```json result.json
response : {
   body : {
    "data": "encrypted-string",
    "isEncrypted": true
   }
}
```
