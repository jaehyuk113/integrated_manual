<Update label="2025-08-06">
  # 리턴 URL 개선

  제출 완료 후 리턴 URL에 submission_id 파라미터가 자동으로 추가되도록 기능이 개선되었습니다. 연동된 파트너 측에서 제출 결과를 보다 안정적으로 추적할 수 있게 되었습니다.

  # 암호화 알고리즘 업데이트

  기존 암호화 방식에 AES-GCM 알고리즘을 추가하고, 이에 대응하는 신규 암호화 키도 적용되었습니다. 관리자 설정에 따라 다양한 보안 모듈을 선택할 수 있도록 확장되었습니다.
</Update>

<Update label="2025-08-01">
  # 수동 제출 제어 시스템

  관리자는 프로젝트에 긴급 중단이 필요하다고 판단 시, 신규로 들어오는 submission을 즉각적으로 막을 수 있습니다. 중단된 프로젝트를 재개하기 위해선, 대시보드에서 변경이 필요합니다.

  # 재시도 플로우 개선

  앞면과 뒷면에서 재시도 메시지가 발생한 경우, 뒷면 페이지부터 다시 찍도록 설정되어 왔습니다. 이제 재시도 메시지가 발생한 경우, 앞면 이미지부터 찍을 수 있도록 변경되었습니다.
</Update>

<Update label="2025-07-30">
  # 제출 속도 제한

  설정 시, 설정한 횟수나 허용된 시간을 기반으로 유저의 진입을 허용합니다. 설정된 값에 위반되는 경우, liveform 접근이 허용되지 않습니다. 또한, 접근차단 시간 설정을 통해, 앞단의 허용 로직에 위반되는 사용자들이 다시 liveform에 접근 가능한 시간을 부여합니다.

  # 암호화된 이미지 전송

  FE에서 이미지 업로드 시 암호화해서 전송하도록 구현되었습니다. GET Submission API에서 id_LivenessScore를 제공합니다.
</Update>

<Update label="2025-07-28">
  # 세로 모드 지원

  Step 1 신분증 촬영 시, 세로 모드 지원 및 UI 변경이 적용되었습니다.

  # 엔진 업데이트

  ID 엔진이 46개국의 95개 신분증을 지원하도록 업데이트되었습니다.

  # 이름 필드 영문화 변환

  실행 시, 신분증 종류 및 국가에 상관없이 모든 신분증에 적힌 이름들이 영문으로 변경되어 대시보드에 저장됩니다.
</Update>

<Update label="2025-07-21">
  # SMS 인증 기능

  암호화된 URL 쿼리를 사용해야 하는 SMS 인증 기능이 추가되었습니다. 개발자 가이드를 참조하세요. 관리자는 제출 후 제출 세부사항에서 전화번호를 확인할 수 있습니다.

  # UX 개선

  Active liveness는 팁 버튼으로 캡처됩니다. Passive liveness는 기존 "셀피 가이드 보기"를 숨기고 팁 버튼으로 캡처됩니다. 중복 사용자 거부 메시지가 "KYC가 거부되었습니다"에서 "귀하의 신원이 이전에 확인되었습니다. 정책에 따라 추가 시도는 허용되지 않습니다"로 업데이트되었습니다.
</Update>

<Update label="2025-07-14">
  # eKYC 프로세스 재개 기능

  사용자는 이제 페이지를 벗어나거나 새로고침하거나 페이지를 재시작한 후에도 Step 1과 Step 2 사이의 중단된 eKYC 프로세스를 재개할 수 있습니다. 시스템은 15분 유효성을 가진 고유한 해시 기반 식별을 사용하여 사용자 진행 상황을 추적하고 적절한 재개 지점을 결정합니다.
</Update>

<Update label="2025-06-27">
  # POST/REVIEW API 추가

  프로젝트 관리자가 pending 상태의 KYC 제출 건을 직접 검수할 수 있는 API가 추가되었습니다. 프로젝트 설정에 따라 검수자를 고객사(Client) 또는 Argos Agent로 지정할 수 있으며, Client로 설정 시 Agent는 해당 pending 건을 검수하지 않습니다.
</Update>

  대시보드 > 설정 > 다중 검증 및 사기 방지에 CURP 검증 기능이 추가되었습니다. 3rd Party API에서 CURP 번호를 검증합니다.

  대시보드 > 설정 > 자금 세탁 방지에 AML 감시 목록 필터링이 추가되었습니다. 관리자는 AML 점수 임계값과 정확한 생년월일 일치를 위한 체크박스를 설정할 수 있습니다.

<Update label="2025-06-16">
  # QueryString 파라미터 selectedIssuingCountry와 selectedIdType 추가 

  선택된 국가와 신분증 유형으로 KYC 프로세스를 바로 시작할 수 있습니다. URL Query에 encrypted 파라미터로 암호화하여야 설정이 가능합니다.
</Update>

<Update label="2025-05-30">
  # 리턴 URL 기능 추가

  관리자는 대시보드에서 사전에 리턴 URL 과 전달할 파라미터 항목을 선택하실 수 있습니다. 제출자는 KYC 프로세스가 완료된 뒤 "Ok" 버튼을 선택 시, 설정된 리턴 URL 과 함께 해당 Submission 의 파라미터로 리디렉션 합니다.
</Update>

<Update label="2025-05-23">
  # 웹훅 Retry 추가

  Submission 진행 간 retry 오류가 발생 시 웹훅이 트리거 됩니다.

  # GET/SUBMISSION 업데이트

  인젝션 이력이 GET/SUBMISSION 에서 조회가 가능합니다. 최대 이력은 최신 건 부터 10건까지 조회가 가능하며, 이력에는 상세 주소 정보는 포함되지 않습니다.
</Update>

<Update label="2025-05-14">
  # Address Injection

  서브미션이 승인된 건의 경우, Injection 기능을 통해 주소 정보를 추가 할 수 있습니다. 이 때 VPN 이나 proxy 와 같이 비정상적인 접근인 경우 혹은 사용자의 IP 주소와 입력한 주소 거리가 먼 경우 진행을 제한시킬 수 있습니다.
</Update>

<Update label="2025-05-08">
  # Custom DI 제공

  관리자는 Custom DI 를 Submission 에서 제공되는 정보를 기반으로 설정할 수 있습니다. 설정된 Custom DI 로 관리자는 중복 관리를 할 수 있습니다. API 으로는 GET/SUBMISSION 을 통해 custom DI 를 조회할 수 있습니다.

  # URL Query 암호화 옵션 allowedIdTypes, allowedDocumentFields 추가

  신분증 기반의 KYC 인 ID document 프로세스가 진행되는 경우, 허용할 신분증 타입과, 허용할 필드를 설정할 수 있습니다. URL Query 에 encrypted 파라미터로 암호화하여야 설정이 가능합니다.

  # Projection

  GET/SUBMISSION 시 노출되는 개인의 민감한 정보들은 Projection 을 설정하여 노출되지 않게 설정하실 수 있습니다. Projection 목록은 GET/Projection, POST/Projection, and DELETE/Projection로 관리하실 수 있으며, Projection 적용은 사용자가 KYC 진행시 입력하는 URL 의 Query String 옵션을(Encrypted 필수) 사용하여 적용하실 수 있습니다.
</Update>

<Update label="2025-03-27">
  # Knowledge-based 추가

  기존의 LiveForm 은 ID Document 로 정의하고, 신분증 없는 신원확인을 위한 Knowledge-based 기능을 추가하였습니다. 관련해서 매뉴얼의 암호화, Query String 관련 업데이트를 하였습니다.
</Update>

<Update label="2024-01-23">
  # DI 필드 추가

  - `GET/Submission API`: 중복 데이터를 식별하고 관리할 수 있는 DI(duplicated information) 필드가 추가되었습니다. 이 필드는 중복된 정보의 해시값을 제공합니다.
  - `Webhook Events`: 모든 이벤트에 DI(duplicated information) 필드가 추가되어, 중복된 정보에 대한 해시값을 반환합니다.
</Update>

  # 개발 문서가 새로워졌습니다 🚀

  더욱 체계적이고 직관적인 구조로 문서를 재구성했습니다.
  필요한 정보를 빠르게 찾아볼 수 있도록 목차와 섹션을 깔끔하게 정리했어요.

  ### 2. API 레퍼런스를 강화했습니다

  이제 모든 엔드포인트에 상세한 설명과 함께 실제 예제 코드를 제공합니다.

  처음 사용하는 사용자도 쉽게 따라할 수 있도록 단계별 가이드를 제공합니다.
  기초부터 시작해 문서를 활용하는 방법을 차근차근 배워보세요.

  ### 4. 검색 기능을 업그레이드했습니다

  원하는 정보를 더 빠르게 찾을 수 있도록 검색 알고리즘을 개선했습니다.
  키워드 자동 완성과 관련 문서 추천 기능도 추가되었어요.
