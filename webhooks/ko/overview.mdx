## 웹훅이란?

웹훅은 특정 이벤트가 발생했을 때 실시간으로 알림을 받을 수 있는 기능입니다. KYC 데이터가 변경되거나 새로운 정보가 추가될 때마다 지정된 URL로 자동으로 알림을 보냅니다.

## 1. 웹훅 이벤트 타입을 알아보세요

ARGOS Identity 웹훅으로 다음과 같은 실시간 업데이트를 받아보세요.

- "webhook_trigger": "created" → 서브미션이 **생성**된 경우
- "webhook_trigger": "retry" → 진행간에 **재시도**가 발생된 경우
- "webhook_trigger": "approved" → 서브미션이 **승인**된 경우
- "webhook_trigger": "rejected" → 서브미션이 **거절**된 경우
- "webhook_trigger": "submit" → 서브미션 상태가 **보류**(pending)된 경우
- "webhook_trigger": "updated" → 서브미션이 **업데이트** 된 경우
- "webhook_trigger": "delete" → 서브미션이 **삭제**된 경우
- "webhook_trigger": "injection" → 서브미션에 **Injection** 이 진행된 경우
- "webhook_trigger": "aml" → **aml** 프로세스가 진행된 경우

웹훅을 활용하면 KYC 프로세스의 모든 단계를 실시간으로 모니터링하고, 필요한 조치를 즉시 취할 수 있어 고객 온보딩 프로세스를 더욱 효율적으로 관리할 수 있습니다.

## 2. 웹훅 이벤트를 등록하세요

<img
  height="200"
  src="/images/webhooks/webhook.png"
/>

<Steps>
  <Step title="대시보드 로그인">
    ID Check [대시보드](https://idcheck.argosidentity.com/)에 로그인합니다.
  </Step>
  <Step title="설정 메뉴 접근">
    대시보드 상단 네비게이션 바에서 `설정` 메뉴를 클릭합니다.
  </Step>
  <Step title="Access Management 페이지 이동">
    좌측 사이드바의 `General` 섹션에서 `Access Management`를 선택합니다.
  </Step>
  <Step title="웹훅 URL 입력">
    `Webhook URL` 섹션을 찾아 입력 필드에 원하는 웹훅 주소를 입력합니다. 입력한 URL은 `HTTPS` 프로토콜을 사용해야 하며, 유효한 형식이어야 합니다.
  </Step>
  <Step title="설정 저장">
    페이지 하단의 `저장` 버튼을 클릭하여 변경사항을 적용합니다.
  </Step>
</Steps>

## 3. 안전한 데이터 전송을 위한 암호화 옵션을 활성화하세요

ARGOS Identity는 웹훅을 통해 전송되는 민감한 KYC 데이터를 보호하기 위해 암호화 옵션을 제공합니다. 이 기능을 활성화하면 모든 웹훅 데이터가 자동으로 암호화되어 더욱 안전한 정보 교환이 가능해집니다.

### a. 암호화된 웹훅 데이터는 어떤 모습인가요?

암호화가 적용된 웹훅 응답은 다음과 같은 형식으로 전달됩니다:

```json data-encrpyted.json
{
  "response": {
    "body": "encrypted-string"
  }
}
```

여기서 `encrypted-string`은 원본 데이터를 암호화한 결과입니다.
